Introduction:
 
The lost and found mobile application is extended to a geo-spatial features, where we are using the google console and adding the google maps SDK and the places API (new) into the system, while keeping all the rest of the functionalities same as the Task7.1P
 
Use case of commercializing the app:
 
The use cases of this app is more in schools, universities, public places, airports, railway stations, shopping malls, events, etc, Since we have a large audience, so utilizing this app for a larger scale base would be a great approach. This would provide a platform for everyone to join and make use of it. Thus, commercializing this application would have profitable results.
 
Requirements:
 
• We also would require robust authentication, need to ensure the security, privacy, etc.
• We also need the cloud data storage, as managing all the location queries in local database would be not sufficient.
• Using computer vision for matching the lost item and found item database, which would reduce the manual verification.
• Email alerts feature can be added by an email integration, such that whenever the user’s lost item is found we can send them an alert notification on email.
• Along with all these features, we can add an in-app messaging feature which allows the user who have lost the item and the user who have found the item to contact so that they can receive their lost item back as per their convenient time and place.
• We can also add “urgency” option in the app, which helps people find their lost item on urgent basis, as we can run ads, or feature their listing on top for 1 week.
